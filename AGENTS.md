# AGENTS.md

## Назначение

Этот файл фиксирует ожидания к агенту, который работает с репозиторием MLX Whisper Dictation.

Проект представляет собой menubar-приложение для macOS на Apple Silicon, которое:

- локально записывает звук с микрофона;
- распознает речь через MLX Whisper;
- пытается вставить текст в активное поле ввода;
- при невозможности автовставки сохраняет распознанный текст в буфер обмена;
- показывает состояние приложения в строке меню.

## Главные требования к приложению

При изменении кода агент должен сохранять и усиливать следующие свойства приложения:

- Приложение должно работать на macOS с Apple Silicon.
- Основной сценарий распространения: сборка `.app` и ручное обновление через перетаскивание `dist/Dictator.app` в `Applications`.
- Интерфейс, уведомления, подписи пунктов меню, комментарии и docstring должны оставаться на русском языке.
- Приложение должно быть menubar-приложением без отдельного основного окна.
- Распознавание должно выполняться локально, без облачных сервисов.
- При неудачной автовставке результат не должен теряться: текст обязан оставаться в буфере обмена.
- В меню должны быть видны хотя бы следующие статусы:
  - текущая модель;
  - текущий хоткей;
  - максимальная длительность записи;
  - статус приложения: ожидание, запись, распознавание;
  - статус необходимых разрешений macOS.
- Изменения не должны ломать сценарий записи из menu bar даже если глобальный хоткей временно не работает.

## Главные требования к агенту

Агент, работающий с этим репозиторием, должен соблюдать следующие правила:

- Писать ответы, документацию, логические подписи в UI и developer-facing текст на русском языке.
- Перед правками сначала проверять текущий код и не предполагать, что установленная в `Applications` копия совпадает с кодом в репозитории.
- Помнить, что пользователь обновляет приложение вручную через drag-and-drop из `dist/Dictator.app` в `Applications`.
- Не полагаться на LaunchAgent как на основной способ запуска приложения.
- Предпочитать минимальные и локальные изменения вместо большой перестройки архитектуры.
- Сохранять один основной runtime-entrypoint в файле `whisper-dictation.py`, если пользователь явно не попросил другого.
- При проблемах с хоткеями или автовставкой сначала проверять слой разрешений macOS (`Accessibility`, `Input Monitoring`, `Microphone`) и только потом подозревать модель распознавания.
- При изменениях, влияющих на сборку или запуск, обновлять документацию в `README.md`.
- Для локальной проверки предпочитать:
  - `uv sync --dev`
  - `uv run ruff check .`
  - `python setup.py py2app -A`
- Не навязывать standalone-сборку как единственный путь, если alias-сборка решает пользовательскую задачу.

## Что агент должен проверять при регрессиях

Если пользователь сообщает, что приложение перестало работать, агент должен последовательно проверить:

1. Какой именно `.app` реально запускается.
2. Актуальны ли логи в `~/Library/Logs/whisper-dictation/`.
3. Какие разрешения видит приложение и что реально включено в `System Settings`.
4. Сохраняется ли текст хотя бы в буфер обмена.
5. Работает ли запись из menu bar независимо от хоткея.
6. Не сломан ли сценарий обновления через замену `.app` в `Applications`.

## Команды по умолчанию

Базовые команды для агента в этом репозитории:

```bash
uv sync --dev
uv run ruff check .
uv run python whisper-dictation.py
uv run python setup.py py2app -A
```

## Логи и диагностика

Если нужно диагностировать runtime-проблемы, агент должен помнить про файлы:

```bash
~/Library/Logs/whisper-dictation/stdout.log
~/Library/Logs/whisper-dictation/stderr.log
```

Если после обновления `.app` хоткеи перестают работать, агент должен рассматривать это как вероятную проблему пересогласования прав macOS, а не только как ошибку в коде.
